# AI SkillHub Skill Router Standard

This document is the stable rule for parent/child Skill routing in AI SkillHub.

## Markers

- `[ROUTER-HUB]`: a parent Skill generated by AI SkillHub for a repository or collection that contains multiple child Skills.
- `[CHILD-SKILL]`: a focused child Skill from the source repository.

## Rules

1. AI SkillHub must generate parent router Skills outside the author's source repository, under `app/github_sources/AI-SkillHub-local-routers`.
2. AI SkillHub must not edit author-owned `SKILL.md` files to add parent/child markers.
3. Parent router names must stay callable by the original collection name, such as `/nature-skills`.
4. Child Skill names must stay callable by their original names, such as `/nature-figure`.
5. The router file must include `[ROUTER-HUB]` in both frontmatter description and the first heading.
6. The frontmatter `description` value must be quoted when it starts with `[ROUTER-HUB]`, otherwise YAML parsers can treat the marker as a flow sequence and skip the Skill.
7. The router child list must prefix every child entry with `[CHILD-SKILL]`.
8. Daily auto-update and manual sync must regenerate routers after source updates, so new repositories and new child Skills inherit the same standard automatically.

## Why The Markers Are ASCII

The router marker is intentionally ASCII-only. This avoids mojibake across Windows PowerShell, Git, Codex Skill parsing, and copied folders on different machines.

UI surfaces may translate these markers into Chinese labels, badges, or icons, but the underlying `SKILL.md` files should keep `[ROUTER-HUB]` and `[CHILD-SKILL]` unchanged.
