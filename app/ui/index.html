快速上手

AI SkillHub 应该怎么用？

它不是内置 Skill 商店，而是帮你把 GitHub、zip、本地文件夹里的真实 Skill 统一管理，然后链接给已安装的 AI Coding 工具。

来源

只有目录里有 SKILL.md，才会被当作 Skill。Prompt 仓库会保留为资料，不会强行安装。

工具

没安装 Codex 或 Antigravity 不算错误。AI SkillHub 会跳过缺失工具，不创建假目录。

分享

给别人前运行“分享前检查”。如果出错，让对方导出诊断包发回来。
AI 软件详情

AI 工具接管状态

这里会把检测、目录权限、接管方式和下一步建议分开说明，避免把“检测到目录”和“已经接管”混在一起。
发布中心

开发者与发布检查

这些动作只用于开发、分享前验收和发布前检查。正式上传 GitHub Release 前，先让这里全部通过。

系统体检

重新生成当前电脑的诊断报告，确认 Git、WebView2、配置、skills 和 AI 工具状态。

分享验收

模拟干净下载用户、中文路径、缺 Codex、无 AI 工具、缺 Git 和缺 WebView2。

排错包

导出脱敏报告包，方便别人电脑出错后发回来分析。

发布预检

生成白名单 zip 和 SHA256，并检查包内没有个人 skills、来源、配置、报告或缓存。
