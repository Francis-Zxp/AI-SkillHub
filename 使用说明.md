# AI SkillHub 使用说明

当前版本：`v1.1.1`

AI SkillHub 用来把多个 GitHub Skills 仓库集中管理到一个共享目录，并自动接管 Claude Code、Codex、Antigravity 能识别的技能入口。

当前版本的界面使用 WebView2：外层仍是 `AI SkillHub.exe`，但界面由 `app\ui` 里的 HTML/CSS/JS 渲染，避免旧式 Windows 控件出现黑框、乱码和错位。

公开发布版只包含 AI SkillHub 程序，不包含任何人的个人 `skills`、`app\github_sources`、诊断包或本机配置。第一次运行时会自动生成空白 `app\skillhub.config.json`，每个人再按自己的需要添加 GitHub 来源。

## 日常怎么用

1. 双击 `AI SkillHub.exe`。
2. 第一次打开时先看顶部“新用户引导”，按 `添加来源 -> 同步 -> 接管 AI 软件链接` 三步走。
3. 点击 `立即同步`，软件会更新 `app\github_sources` 里的 GitHub 仓库，并重建 `skills` 里的链接。
4. 打开“每日自动更新”开关后，Windows 会每天 09:00 自动同步。
5. 打开“接管 AI 软件链接”开关后，Claude Code、Antigravity 会指向共享 `skills` 文件夹；如果当前电脑安装过 Codex，Codex 会在自己的 skills 目录下建立每个 Skill 的单独链接。没安装 Codex 时会自动跳过，不影响 Claude 使用。
6. 右上角可以切换主题：默认是“可爱”，也可以切到“清新”或“深色”。

左侧新增“快速上手”按钮，适合刚分享给别人时让对方快速理解来源、Skill、Prompt 和 AI 工具接管的关系。点击左侧大 Logo 会触发一个轻量的小动画。技能列表和仓库列表的表头可以拖动分隔线调整列宽，适合你自己决定“名称、分类、说明、来源位置”各占多少空间。

## 检测、接管和链接是什么意思

左侧“系统体检”卡片里新增了“AI 软件详情”。点击后可以看到每个工具的检测状态、接管状态、目录权限、目录位置和下一步建议。

- `已检测`：说明电脑上存在该工具的配置目录或 skills 目录，例如 `~\.codex\skills`。
- `尚未接管`：说明 AI SkillHub 还没有把共享 `skills` 接到该工具里，通常是还没有打开“接管 AI 软件链接”，或还没有执行同步。
- `已接管 X 个 Skill`：常见于 Codex。Codex 不是把整个 `skills` 根目录变成一个链接，而是在 `~\.codex\skills` 下面给每个 Skill 建单独链接，所以会显示接管数量。
- `可写`：说明该目录权限正常，软件有能力在里面建立链接或复制文件。

所以 `OpenAI Codex - 已检测 - 尚未接管 · 可写 · ~\.codex\skills` 的意思不是报错，而是“Codex 目录存在、权限正常，只是还没有由 AI SkillHub 接管”。

## 列表搜索、筛选和启用状态

`v1.0.5` 开始，列表区分为三页：

- `已启用技能`：当前 AI 软件可以读取到的 Skill。
- `仓库来源`：所有 GitHub / 本地来源，包括 Skill 和 Prompt 资料。
- `Prompt 资料`：只保存润色提示词或参考资料，不安装为 Skill。

列表右侧可以搜索、按分类筛选、按名称/分类/仓库/类型/健康状态排序。仓库来源页的“状态”列有一个小开关：关闭后该来源会保留在来源目录和配置中，但不会继续安装为 Skill；再次打开会重新参与同步。Prompt 来源默认只作为资料保存，不提供启用开关。

列表下方还有一排场景预设：`全部`、`论文`、`图表`、`UI 设计`、`安全`、`Prompt`。这些按钮会自动切到对应标签页并筛选相关分类，适合 Skill 很多时快速找到当前任务需要的能力。

## 标签、重复分组和 Prompt 详情

`v1.0.5` 第三阶段开始，添加或编辑来源时可以填写“标签”，多个标签用逗号分开，例如：`论文, 常用, 图表`。

标签会保存到 `app\skillhub.config.json`，并跟随来源显示在 Skill、仓库来源和 Prompt 资料列表里。列表摘要区会显示当前页的标签云，点击某个标签会进入真正的标签筛选状态，不会占用搜索框。也可以点“全部标签”恢复完整列表，或点“未标签”找出还没有整理标签的来源。

列表还会显示“重复技能”分组。如果发现同名 Skill 或同名来源，会以 `名称 × 数量` 的方式列出来，点击后可以直接定位。这样以后 Skill 越来越多时，可以先看哪些能力可能重复，再决定是否保留、停用或删除来源。

当你在“仓库来源”或“Prompt 资料”里点击某一行，下方会出现详情面板。Prompt 详情会展示名称、分类、健康状态、地址、备注和标签；如果要改分类、备注或标签，直接在上方编辑区修改后点“保存修改”即可。详情面板里的“打开此来源”可以直接打开该来源的本地文件夹。

## 最近操作时间线

底部“运行反馈”区域现在会显示最近操作时间线，包括同步、系统体检、zip 导入预览测试、本地导入、添加/保存/删除来源、启用/停用来源等。  
这不是临时弹窗，而是从 `app\reports\operation-history.jsonl`、`last-sync.md`、诊断报告和 zip 预览测试报告里汇总出来的真实记录。后续别人电脑出问题时，可以先看最近做过什么，也可以点击“导出排错包”把这些报告合成一个脱敏 zip。

## 添加新的 GitHub 项目

推荐在软件里粘贴 GitHub 地址并点击“添加”。  
软件只接受 `https://github.com/作者/仓库.git` 或 `https://github.com/作者/仓库` 这种普通仓库地址，避免误执行奇怪链接。软件会自动修正复制粘贴时常见的空格问题，例如把 `https: //github.com/...` 修正成 `https://github.com/...`。

也可以手动把仓库克隆到 `app\github_sources`。只要仓库里有 `SKILL.md`，下次同步会自动发现；如果以后删除了这个仓库，下次同步也会删除不再存在的旧链接。

## 导入本地文件夹或 zip

如果你从 GitHub 下载的是 zip，或者已经把别人的 Skill 项目解压到本地，可以在“添加仓库”区域点击“选择文件夹”或“选择 zip”。

软件会先做导入预览，告诉你：

- 里面发现了多少个 `SKILL.md`。
- 是否更像 Prompt / 参考资料，而不是可安装 Skill。
- 推荐导入名称和示例 Skill。

确认没问题后，再点击“导入到来源并同步”。如果没有发现 `SKILL.md`，软件会按 Prompt / 参考资料保存，不会误装成 Skill。

如果推荐名称已经存在，软件会自动改成 `名称-2`、`名称-3` 这类安全名称，不会直接覆盖旧来源。每次本地导入都会记录到 `app\reports\import-history.jsonl`，里面包含导入时间、来源、目标位置和撤回提示。

开发和排错时可以运行 `AI SkillHub.exe --zip-preview-test` 做 zip 导入预览自动测试。它会临时生成一个示例 zip，确认软件能识别 `SKILL.md`，并验证带危险路径的 zip 会被拒绝。测试报告保存在 `app\reports\zip-preview-test`，不会写入正式 `skills` 或 `app\github_sources`。如果已经跑过该测试，后续导出的诊断包也会带上最新 zip 预览测试报告。

## 自动分类与手动备注

自动分类会读取每个 `SKILL.md` 的名称和说明，判断它更像论文科研、科研图表、界面设计、安全审计等类型。  
如果你更清楚某个仓库的用途，可以在添加仓库时手动选择“细分分类”，并在“手动备注”里写用途。备注会显示在仓库列表和技能列表里。

## 仓库来源健康状态

“仓库来源”列表现在有“健康”列，用来判断每个来源是否真的能启用：

- `可用 / 数量 Skills`：已经发现可安装的 `SKILL.md`。
- `资料保留`：这是 Prompt / 参考资料来源，不会安装为 Skill。
- `无 SKILL.md`：文件夹存在，但暂时没有发现可安装 Skill。
- `未下载`：配置里有这个来源，但本地文件夹不存在，需要重新同步。
- `本地`：来自手动导入或本地文件夹，不一定能自动 `git pull` 更新。

## 目录结构

根目录保持简洁：

- `AI SkillHub.exe`：主程序
- `skills`：AI 软件实际读取的技能入口
- `使用说明.md`：这份说明
- `app`：程序脚本、WebView2 界面、GitHub 来源、报告、图标和必要运行包

## 分享给别人或换电脑

把整个 `AI_global_skills` 文件夹复制到新电脑后，先双击 `AI SkillHub.exe`。程序会检查当前所在路径，如果发现文件夹位置变了，会自动用新位置重新同步一次，并按配置重新接管 Claude Code、Antigravity，以及已安装的 Codex 技能入口。

对方电脑需要已经安装 Git 和 WebView2 Runtime。大多数 Windows 11 电脑已经自带 WebView2；如果第一次打开失败，优先检查这两个环境。

`v1.1.0` 新增了“分享前检查”。准备把文件夹发给别人之前，可以先点左侧的“分享前检查”，软件会模拟“对方电脑只有 Claude / Claude Code、没有 Codex”的情况生成诊断包。这个检查通过时，Codex 未检测到会被写成“可忽略信息”，不会被当成程序失败，也不会创建假的 Codex 目录。

如果对方电脑没有 Codex，只要 Claude / Claude Code 已安装，并且 Git、WebView2 正常，AI SkillHub 就应该可以继续使用 Claude 的 Skills 接管能力。Codex 以后安装了，再打开“接管 AI 软件链接”即可。

如果对方电脑暂时没有安装 Claude Code、Codex 或 Antigravity，AI SkillHub 会提示“未识别到可接管的 AI Coding 工具”。这种情况不是程序坏了，而是当前没有可接管入口；安装任意一种支持的 AI Coding 工具后，再重新打开接管开关即可。AI SkillHub 不会为了通过检查而创建假的工具目录。

## 排错包和诊断包怎么选

如果你只是想看当前电脑环境是否正常，点击“导出诊断包”或“立即体检”。它会检查 Git、Node.js、WebView2、配置文件、skills 目录和 AI 工具目录。

如果别人电脑已经出错，或者你要把问题发给开发者看，优先点击“导出排错包”。排错包会把最近操作记录、本地导入记录、同步报告、最新诊断报告和 zip 导入预览测试报告合成一个 zip，并做基础脱敏：AI SkillHub 根目录会替换成 `{AI_SKILLHUB_ROOT}`，用户目录会替换成 `{USERPROFILE}`，其它 Windows 绝对路径会替换成 `{WINDOWS_PATH}`。

排错包不会包含真实 `skills` 内容、`app\github_sources` 仓库源码、WebView2 缓存或发布包。开发和验证时可以运行 `AI SkillHub.exe --troubleshooting-test`，确认排错包能正常生成。

## 可以添加的重要来源示例

这些来源只是示例，不会随公开仓库一起打包。你可以在自己的电脑上按需添加：

- `garden-skills`：网页级交互界面与设计工程参考。
- `ui-ux-pro-max-skill`：界面设计、设计系统、可用性和动效参考。
- `claude-code-best-practice`：Claude Code 最佳实践技能。
- `VibeSec-Skill`：安全检查参考，用于限制脚本执行路径、GitHub 地址格式和本地文件访问范围。

## 图标与界面

程序图标来自 `app\assets\AI SkillHub.ico`。界面主 Logo 使用 `app\assets\AI SkillHub.logo.ui.png`，这是从原始 Logo 生成的小尺寸 UI 专用图，启动时会随 HTML 首屏直接加载，避免先显示文字再显示图标。如果 Windows 任务栏短时间内还显示旧图标，通常是系统图标缓存或旧快捷方式导致，重新固定新的 `AI SkillHub.exe` 即可。

## 编码规则

配置文件内部不再保存中文分类，而是保存英文分类 ID，再由界面翻译成中文、英文或韩文。脚本和报告统一写入 UTF-8 BOM，避免 Windows PowerShell 再次把中文写成乱码。
## v1.0.4 系统体检

首页左侧现在会显示“系统体检”卡片，添加仓库区域支持本地文件夹 / zip 导入预览，仓库来源列表也会显示来源健康状态。

系统体检包括：

- Git、Node.js、WebView2、配置文件、skills 目录等关键检查。
- Claude Code、Codex、Antigravity 的检测状态、skills 目录是否存在、是否已链接、是否可写。
- 最近一次诊断报告的正常、提醒、错误数量。

点击“立即体检”会重新生成诊断报告，并刷新首页体检状态。Node.js v24 LTS 会被视为适合后续前端/Tauri 准备工作的版本。

体检卡片现在显示四类数量：正常、提醒、错误、信息。`信息` 通常表示“不是故障，只是需要知道的情况”，例如某台电脑没有安装 Codex，或者第三方 Skill 内容里有需要人工阅读的安全提示。

安全扫描现在会区分两类提示：

- `程序自身风险提示`：AI SkillHub 自己的代码是否存在可疑执行、危险删除、泄露密钥等问题。
- `第三方 Skill 内容提示`：安装的 Skill 文档、示例脚本或参考资料里出现的可疑模式。这需要人工阅读，但不等于 AI SkillHub 程序本身有漏洞。

如果对方电脑只安装 Claude / Claude Code，没有安装 Codex，诊断会显示 Codex 未检测到，但不会创建假的 Codex 目录，也不会把整体状态判定为错误。

## v1.0.3 诊断包

如果软件在别人电脑上运行失败，先让对方打开 AI SkillHub，点击左侧的“导出诊断包”。

软件会生成三类文件：

- `app/reports/latest-diagnostics.json`：给 Codex/Claude 读取的机器状态。
- `app/reports/diagnostics/skillhub-diagnostics_时间.md`：普通人能读的检查报告。
- `app/reports/diagnostics/skillhub-diagnostics_时间.zip`：发给开发者排错的压缩包。

诊断只检查环境和导出报告，不会自动创建不存在的 Codex、Claude 或 Antigravity 目录，也不会删除用户文件。
