# AI SkillHub 使用说明

当前项目已经整理为当前桌面版。旧版 V1 的 `app` 目录、旧启动器
和旧发布目录已经删除。以后只维护 AI SkillHub 主线。

## 1. 启动

本机直接双击：

```text
D:\My Files\AI_global_skills\AI SkillHub.exe
```

如果以后发布给别人，应使用 AI SkillHub 发布包，而不是直接把你的开发目录整体复制出去。

## 2. 添加新的 Skill 来源

进入 `Sources` 页面：

1. 粘贴 GitHub 仓库地址，或选择本地文件夹、zip、`.skill` 包。
2. 选择类型：`技能 Skill`、`Prompt 资料` 或其它资料。
3. 选择分类、标签和备注。
4. 点击 `一键添加并刷新`。
5. 如果需要让 Claude Code、Codex、Antigravity 直接识别，再打开同步到 AI 工具的授权开关。

普通添加会先进入 AI SkillHub 来源库并刷新索引。写入 AI 工具目录必须经过用户授权。

## 3. 什么会被识别为 Skill

只有包含 `SKILL.md` 的文件夹才会被识别为 Skill。

有些仓库的可安装内容不在根目录。例如 PaperSpine 的真正 Skill 位于：

```text
dist/codex/skills/
dist/claude/skills/
dist/openclaw/skills/
```

AI SkillHub 的扫描规则已经支持这类结构。

## 4. 母 Skill 与子 Skill

多 Skill 仓库会自动生成一个母入口，方便新手不知道具体子 Skill 时先调用母 Skill。

底层标准：

```text
[ROUTER-HUB]   母 Skill / 路由入口
[CHILD-SKILL] 子 Skill / 具体能力
```

生成的母入口位于：

```text
app-next\data\github_sources\AI-SkillHub-local-routers\
```

AI SkillHub 不会修改作者仓库里的原始 `SKILL.md`，所以 GitHub 更新不会覆盖这套规则。

## 5. 同名 Skill 冲突选择

如果两个项目里都有同名子 Skill，例如：

```text
Nature-Paper-Skills / figure-planner
PaperSpine / figure-planner
```

V2 不会偷偷删除、改名、覆盖或自动选一个。进入 `Sources` 页面，可以在“同名 Skill 冲突选择器”里：

1. 查看每个同名 Skill 来自哪个项目。
2. 设置直接调用时的默认来源。
3. 重置为待选择。
4. 忽略提醒。

你的选择会写入本机 V2 SQLite 表 `skill_conflict_choices`。它不会修改作者原始仓库，也不会被 GitHub 更新覆盖。

目前这一步已经完成“检测 + UI 选择 + SQLite 持久化”。后续如果要让 `/figure-planner` 这种实际调用自动读取默认选择，还需要把 Codex/Claude 侧的分发逻辑接入这张表。

## 6. 本地目录说明

```text
AI_global_skills\
  app-next\                         V2 程序源码和运行时
  app-next\runtime\                 V2 辅助脚本
  app-next\data\github_sources\     本机克隆的 GitHub 来源，私有
  app-next\reports\                 本机报告，私有
  app-next\.skillhub-next\          本机同步状态，私有
  skills\                           AI 工具正在读取的共享 Skill 视图，私有
```

`skills` 不是 V1 内容，不能删除。它是 Claude/Codex/Antigravity 实际读取的共享 Skill 目录。

## 7. 分享和 GitHub 发布边界

不要提交或分享这些本机私有内容：

```text
skills\
app-next\data\
app-next\reports\
app-next\.skillhub-next\
app-next\runtime\skillhub.config.json
app-next\node_modules\
app-next\src-tauri\target\
AI SkillHub.exe
```

公开仓库只应包含源码、运行时模板、说明文档和必要资源。

## 8. 开发检查

开发者在提交前建议运行：

```text
cd app-next
pnpm build
cargo test --manifest-path src-tauri/Cargo.toml
pnpm tauri build --no-bundle
```

如果修改了 `app-next\runtime` 里的 PowerShell 脚本，要确保脚本保持 UTF-8 BOM，否则 Windows PowerShell 5.1 可能显示中文乱码。

## 9. 当前状态

- 本地/zip 预览

- 母子 Skill 路由
- 同名子 Skill 冲突检测、选择和本机持久化
- Claude/Codex/Antigravity 链接同步
- 诊断、分享检查、发布预检

剩余重点是继续做 UI 细节、正式发布包体验，以及让 Codex/Claude 侧的实际分发逻辑读取同名 Skill 默认选择。
