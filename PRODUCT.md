# Product

## Register

product

## Users

AI SkillHub is for Windows users who collect many Claude Code, Codex, Antigravity, and local AI-agent Skills but cannot easily see what is installed, what each Skill does, which Agent can call it, or how to keep sources synchronized.

Users are typically managing GitHub Skill repositories, local Skill folders, prompt/reference repositories, and generated parent router Skills from one desktop surface. They need visible status, reliable install/update/delete flows, and clear recovery paths without manually copying files into hidden Agent directories.

## Product Purpose

AI SkillHub visualizes and manages local AI Skills from one place. It should let users import a GitHub or local source, classify and annotate it, scan real `SKILL.md` files, rebuild parent router Skills, resolve same-name child Skill conflicts, and synchronize callable Skills into supported AI Agents.

Success means a new user can download the release package, open the app, add or inspect Skill sources, understand Agent install status, and keep their local Skill hub healthy without editing repository files by hand.

## Brand Personality

Precise, capable, and premium.

The interface should feel like a serious local control center: visually polished enough to create trust and curiosity, but still dense, readable, and efficient for repeated management work.

## Anti-references

- Do not present AI SkillHub as a marketing landing page or oversized hero site.
- Do not hide operational state behind decorative motion.
- Do not remove the right-side drawer/detail workflow that keeps editing stable.
- Do not weaken same-name child Skill conflict handling, parent router Skill generation, or Agent sync visibility.
- Do not ship personal sources, cloned repositories, local reports, diagnostics, caches, or private config in public releases.
- Do not show V2/V3 as the product name; those belong only in version history.

## Design Principles

1. Show the Skill graph directly: sources, parent Skills, child Skills, conflicts, Agent status, and sync health must be visible without hunting through folders.
2. Prefer calm control surfaces over transient popups: editing, conflict review, and diagnostics should stay anchored and recoverable.
3. Make automation explain itself: install, update, delete, sync, router rebuild, and GitHub heat refresh need clear success, deferred, warning, and failure states.
4. Keep motion purposeful: particles, glow, blur, and micro-interactions may add identity, but must never block reading, editing, accessibility, or performance.
5. Keep releases clean and portable: the public package must be path-agnostic and exclude user data.

## Accessibility & Inclusion

Target WCAG AA contrast for normal text and controls. Support reduced motion for particle and transition effects. Avoid color-only status cues by pairing colors with labels or icons. Keep Chinese, English, and Korean labels native-sounding, concise, and consistent across navigation, actions, states, and error messages.
